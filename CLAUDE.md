# Room Rental ŌĆö Design System Repository

## Purpose

This repository is the **single source of truth** for all UI/UX standards in the Room Rental Management System.
It is consumed by every frontend application in the project.

## Repository Structure

```
tokens/tokens.json              ŌåÉ W3C Design Tokens ŌĆö the only file humans edit for token changes
generated/                      ŌåÉ auto-generated by scripts/generate-tokens.js ŌĆö never edit manually
  web/tokens.css                ŌåÉ CSS custom properties
  blazor/RoomRentalTheme.cs     ŌåÉ MudBlazor 9.x theme
  mobile/tokens.json            ŌåÉ flat key-value for React Native / Flutter
components/
  specs/                        ŌåÉ markdown component specs (output from Claude Design)
  prototypes/                   ŌåÉ Claude Design HTML exports
guidelines/                     ŌåÉ human-readable design rules
scripts/generate-tokens.js      ŌåÉ Node.js generator, no external deps
```

## Claude Design

Component specs in `components/specs/` and prototypes in `components/prototypes/` are produced by Claude Design (not by any local agent).

When specifying components or screens:
- Always reference token names from `tokens/tokens.json`, never raw hex or px values
- Use `color.primary.DEFAULT`, `spacing.4`, `border.radius.md` notation in specs
- Map to CSS variable `var(--color-primary-DEFAULT)` or MudBlazor `Color.Primary` as appropriate

## Token Naming Convention

Tokens follow dot-notation in `tokens.json`, converted to kebab-case for CSS variables:

| tokens.json path | CSS variable | C# (MudBlazor) | Mobile JSON key |
|---|---|---|---|
| `color.primary.DEFAULT` | `--color-primary-DEFAULT` | `Primary = "..."` | `colorPrimaryDEFAULT` |
| `spacing.4` | `--spacing-4` | padding: `16px` | `spacing4` |
| `border.radius.md` | `--border-radius-md` | `DefaultBorderRadius` | `borderRadiusMd` |

## Workflow

1. Edit `tokens/tokens.json` for any design token change
2. Run `node scripts/generate-tokens.js` ŌĆö regenerates all `generated/` files
3. Commit `tokens.json` + all `generated/` files together in one commit
4. App teams update their reference to pick up the new tokens

## Applications

| Repo | Consumes | Integration point |
|------|---------|-------------------|
| room_rental.auth | `RoomRentalTheme.cs` | `RoomRentalTheme.Create()` in Program.cs |
| (future apps) | `tokens.css` or `tokens.json` | Import in global stylesheet or theme provider |

## Rules for This Repo

- Never edit files under `generated/` manually ŌĆö always regenerate from `tokens.json`
- Never hardcode hex colors or pixel values in component specs ŌĆö use token names
- Every breaking change (token renamed or removed) must bump the major version in `$metadata.version`
- Add a CHANGELOG entry for every merge to main
